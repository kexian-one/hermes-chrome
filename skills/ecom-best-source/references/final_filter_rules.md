# 最终筛选规则

最终筛选策略以原 `ecom-best-source` skill 为基础，但按当前决策移除发票能力、回头率、响应率三项。`huoyuan` 只能作为前置召回、字段补全、API/MCP 替代参考，不能改变最终决策口径。

## 最终打分维度

满分 100，权重固定。原权重保留价格和综合服务分的相对比例 `50:20`，移除其余三项后归一化:

| 维度 | 权重 | 归一化 |
|---|---:|---|
| 价格 | 71.43% | `(最高价 - 当前价) / (最高价 - 最低价) * 100`，越便宜分越高 |
| 综合服务分 | 28.57% | `compositeScore * 20`，5 分制映射到 100 分 |

字段名兼容:

- 价格: `unitPrice` / `unit_price` / `price`
- 综合服务分: `compositeScore` / `composite_score` / `serviceScore`
- 回头率、响应率、发票字段不再作为顶层筛选字段输出，也不参与最终打分；如果接口原始 `detail` 或 `seller_info` 内自带这些信息，可以作为原始详情留存。

## 硬筛和降级

- 品牌不匹配 → 不进入最终 CSV。
- 品类不匹配 → 不进入最终 CSV。
- 有目标规格/SKU 时，`SKU不一致` → 不进入最终 CSV。
- 价格缺失、价格不低于京东价、最终综合得分 `<60` → 不进入最终 CSV。
- 详情接口明确总库存为 0、或目标 SKU/全部 SKU 明确库存为 0 → 不进入最终 CSV。
- 详情接口没有返回库存字段时可保留为“库存待确认”，但写 CSV 前应优先对最终候选补拉一次详情。
- 综合服务分 `< 3.0` → 不推荐，最终得分置 0。
- 入驻年限 `< 1 年` → 不推荐，最终得分置 0。
- `MOQ > 京东购买倍数 * 10` → 标记 `MOQ 过高`，不直接清零。

## SKU 匹配

SKU 匹配仍沿用原 skill 的展示口径:

- `完全一致`: 目标 SKU/规格/数量完全匹配。
- `规格同数量不同`: 规格一致但包装数量不同，可按单件价折算。
- `SKU不一致`: 只能作为 rejected 参考，不进入最终 CSV。

SKU 匹配等级不参与最终权重，但必须输出，供人复核。

## 推荐档位

- `>=80`: 首选
- `70-79`: 次选
- `60-69`: 备选
- `<60`: 不推荐

## 输出策略

- 输出 Top 6。
- 不足 5 条标记“召回不足”。
- 0 条标记“无供给”。
- 保留 rejected 原因和 `score_breakdown`，方便复核为什么被剔除。
