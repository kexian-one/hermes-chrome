# 08 — 技术栈选型 + 运营成本估算

## 技术栈(推荐组合)

### 核心组件

| 层 | 选型 | 替代选项 | 为什么 |
|---|---|---|---|
| 语言 | **Python 3.11+** | Node.js / Go | LLM SDK 生态最完整,异步 IO 成熟 |
| LLM SDK | **`openai>=1.50` 官方 SDK** | LiteLLM | 全 provider 都 OpenAI 兼容,一个 SDK 走天下 |
| 异步 IO | **`asyncio`** | trio | Python 标准,生态广 |
| MCP 协议 | **`mcp` 官方 SDK(Python)** | 自写客户端 | Aggregator 既要 server 又要 client |
| HTTP 服务 | **FastAPI** | Flask / Sanic | 异步原生,Pydantic 集成 |
| 任务队列 | **Redis Streams + redis-py asyncio** | RabbitMQ / Kafka | 简单、消费组、持久化够用 |
| 持久化(任务/状态) | **SQLite**(MVP)/ **Postgres**(规模化) | MongoDB | 文件级 SQLite 起步,后切 Postgres |
| 知识检索 | 阶段 1: **Grep + 文件名**;阶段 2: **Qdrant / Chroma** | 自托管 ES | 起步够用,大了再上向量库 |
| Git 操作 | **GitPython** | shell out | 跨平台稳定 |
| Pydantic | **Pydantic v2** | dataclass | 配置 / 数据模型 |
| 飞书 SDK | **lark-oapi(官方)** | 自己写 webhook | 鉴权/事件订阅都封装好了 |
| 钉钉 SDK | **dingtalk-stream** | 自己写 webhook | 同上 |
| 重试装饰器 | **tenacity** | 自己写 | 久经考验 |
| 日志 | **structlog**(结构化) | logging | jsonl 输出方便后续 LLM 萃取 |
| 配置 | **YAML + pydantic-settings** | TOML / JSON | 人类可读,支持 env var 替换 |
| Web Dashboard | **FastAPI + HTMX / Vue3** | Next.js | 后端 Python 一体化,前端轻 |
| 容器化(可选) | **Docker Compose** | K8s | 单机/小集群够用,不上 K8s |
| 监控(可选) | **Prometheus + Grafana** | 自建 dashboard | 规模化才上 |
| 部署 | **systemd 服务** | Docker / nohup | Windows 服务用 NSSM / Windows Service |

### 关键的不选

- ❌ **LangChain**:过度封装,坑多,LLM 调用透明度差
- ❌ **AutoGen / CrewAI**:多 agent 框架,我们用不到那么花哨
- ❌ **OpenAI Assistants API**:stateful 私有,不兼容其他 provider
- ❌ **Playwright / Selenium**:被 1688 风控(已验证,见 `00-context-and-goals.md` 硬约束 C1)
- ❌ **Anthropic SDK**:用户明确不用 Claude
- ❌ **自己写 MCP 协议**:有官方 SDK,没必要重新发明

## 月运营成本估算

> **关键提示**:以下成本估算基于 **2026 年中** 的市场价格预估,**接入前请到各 provider 官网核当前定价**。不要依赖此处数字做财务决策。

### 单台 PC 的成本

#### 代理 IP(6 个,刚性需求)

| 选项 | 月成本(粗估) | 备注 |
|---|---|---|
| 国外静态住宅(BrightData / Smartproxy / Soax) | $30-100/IP × 6 = $180-600/月 | 适合海外业务 |
| 国内静态住宅(芝麻代理 / 青果代理 / 黑洞代理 等) | ¥30-100/IP × 6 = ¥180-600/月 | **1688 场景优先** |
| 动态代理(按流量) | 看流量,可能更便宜 | 适合低频用,但 1688 频换 IP 反而异常 |

**推荐**:**国内静态住宅 ¥180-600/月**(具体看代理商定价,自己核)。

#### LLM API 调用

按一个月跑 4 次完整 batch(20 账号 × 65 商家 ≈ 5200 单次催单),每次 ~10K input + 1K output token:

**全 DeepSeek V3**(假设最便宜):
- input: 5200 × 10K = 52M token × ¥1-2/M = ¥52-104/月
- output: 5200 × 1K = 5.2M × ¥2-4/M = ¥10-21/月
- **小计: ¥62-125/月**(估算)

**全 GPT-4o**(质量最高):
- input: 52M × $5/M = $260
- output: 5.2M × $20/M = $104
- **小计: $364/月 ≈ ¥2600/月**

**混合路由(推荐)**:
- 70% 任务走 DeepSeek = ¥45-90
- 25% 走 Qwen3-Max 中端 = ¥40-100
- 5% 走 GPT-4o(关键视觉/总结) = ¥130
- **小计: ¥215-320/月**

## 人员成本

按 4 台 PC 算:
- **4 人值守**(每机 1 人盯滑块)
- 每人值守工作:**不全职**,batch 时段集中(每周/月)
- 一次完整 batch ~ 2-3 小时密集 + 间歇查看
- 4 人 × 几小时/周 = 几乎可以兼职做

**如果是请人专职**:¥3000-5000/人/月 × 4 = ¥12000-20000/月。**但通常这种规模建议雇佣兼职 / 内部员工兼任,实际不增加纯人工成本**。

## ROI 评估框架

```
月收益 = (开票率提升 × 涉及发票金额 × 可抵扣税率)
       + (节约的人工催单工时 × 人工时单价)

月成本 = 运营成本(¥1275-4000) + 一次性工程成本/12 个月摊销

ROI = 月收益 / 月成本
```

**示例(假设)**:
- 假设你每月发出去 ¥10万 的发票,过去开票率 60%,现在 90%
- 多开 30% × ¥10万 = ¥3万 多了能抵扣的进项税
- 进项税抵扣 13% ≈ ¥3900 / 月
- 节约人工催单 50 小时/月 × ¥50/小时 = ¥2500
- **月收益 ¥6400**
- 月成本(假设满配) ¥4000
- **ROI > 1.5,有意义**

**如果开票率提升只有 5-10% / 涉及金额小**:
- ROI < 1,**别上这个工程**,继续手动

**关键**:**先跑 Phase 0,数据说话**。

## 成本控制建议

### 1. LLM 成本控制

- 默认走 DeepSeek / Qwen(国产)
- 视觉任务走 Qwen-VL,关键场景才用 GPT-4o
- **Token budget per task**:超了报警 / 切便宜模型
- 缓存重复 prompt(同一 skill 内容 多 task 复用)

### 2. 代理成本控制

- 起步只买 6 个 IP(单机 MVP),验证再扩
- 同 IP 长期复用(不频繁换),静态比动态便宜
- 看代理稳定性,**不稳的便宜 IP 反而浪费 LLM token**(重试多)

### 3. 硬件成本控制

- 用现有 PC,不新买
- 6 浏览器 PC 不需要顶配,**16GB / 中端 CPU 足够**
- 不上服务器/云,**家用机器够用**

### 4. 开发成本控制

- 别一上来 Phase 5,**严格逐 Phase 推进**
- 团队工程师 review 严格,**Phase 1 demo 不达标就别走 Phase 2**
